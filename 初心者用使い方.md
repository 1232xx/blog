# 初心者用使い方

このテーマでブログを運用するときの、いちばん基本的な使い方をまとめたメモです。

## まず覚えること

- 記事は `src/content/posts/` に置く
- 新しい記事は `pnpm new "タイトル"` で作れる
- 下書きにしたいときは、ファイル名の先頭を `_` にする
- 記事を消したいときは、その記事ファイルを削除する

## 新しい記事の書き方

### いちばん簡単な方法

新しい記事はこのコマンドで作れます。

```bash
pnpm new "はじめての記事"
```

すると `src/content/posts/はじめての記事.md` のような記事ファイルが作られます。

### 作られる中身

記事ファイルの先頭には、こういう情報が入ります。

```md
---
title: 'はじめての記事'
pubDate: '2026-07-01'
---
```

このテーマでは、最低限 `title` と `pubDate` が必要です。

### 本文の書き方

frontmatter の下に、普通の Markdown で本文を書きます。

```md
---
title: 'はじめての記事'
pubDate: '2026-07-01'
---

こんにちは。これが最初の記事です。

## 見出し

本文はこの下にどんどん書けます。
```

## 記事の増やし方

記事を増やす方法は 2 つあります。

### 方法 1: コマンドで作る

```bash
pnpm new "新しい記事"
```

おすすめはこの方法です。日付が自動で入るので、最初に迷いにくいです。

### 方法 2: 手でファイルを作る

`src/content/posts/` に `.md` または `.mdx` ファイルを新しく作ります。

例:

- `src/content/posts/hello-world.md`
- `src/content/posts/my-note.mdx`

手で作るときも、先頭に frontmatter が必要です。

## 下書きにする方法

公開したくない記事は、ファイル名の先頭に `_` を付けます。

例:

- `_draft.md`
- `_memo.mdx`

このテーマでは、`_` で始まる記事は一覧にも個別ページにも出ません。

### 下書きの作り方

```bash
pnpm new "_下書きの記事"
```

これで `_下書きの記事.md` のような下書きが作られます。

### 公開に戻す方法

ファイル名の先頭の `_` を消します。

例:

- `_draft.md` -> `draft.md`

## 記事の消し方

### 1. 該当ファイルを削除する

消したい記事が `src/content/posts/my-post.md` なら、そのファイルを削除します。

削除後、その記事は一覧や個別ページから消えます。

### 2. いったん非公開にしたいだけなら

すぐ消さずに隠したいなら、ファイル名の先頭に `_` を付けます。

例:

- `my-post.md` -> `_my-post.md`

これならファイルは残したまま、公開だけ止められます。

## よくある失敗

- ファイルを `src/content/posts/` 以外に置いてしまう
  - 記事として認識されません
- `title` を入れ忘れる
  - ビルドでエラーになることがあります
- `pubDate` を入れ忘れる
  - 同じくエラーの原因になります
- ファイル名が `_` 始まりのまま
  - 下書き扱いになって公開されません

## 画像を入れたいとき

記事の近くに画像を置いて参照できます。

例:

```md
![説明画像](./_assets/example.webp)
```

画像は記事ごとに `src/content/posts/_assets/` のような場所へ置くと分かりやすいです。

## MDX を使いたいとき

`.mdx` にすると、Markdown の中で Astro コンポーネントも使えます。

例:

```mdx
---
title: 'MDX の記事'
pubDate: '2026-07-01'
---

import Callout from '@/components/examples/Callout.astro'

<Callout />
```

## 迷ったらこの順番

1. `pnpm new "記事タイトル"` で記事を作る
2. 本文を書く
3. `pnpm dev` で表示を確認する
4. 問題なければ `pnpm build`
5. 消したい記事はファイルを削除する

## 更新した内容を GitHub に送る

記事を書いたり、設定を変えたりしたら、最後に GitHub に送ります。

### 1. 変更されたファイルを見る

```bash
git status
```

### 2. 変更を追加する

全部まとめて送るなら:

```bash
git add .
```

特定のファイルだけ送りたいなら、そのファイル名だけ指定します。

```bash
git add src/content/posts/my-post.md
```

### 3. コミットする

```bash
git commit -m "Add new post"
```

### 4. GitHub に送る

初めて送るときはこれで大丈夫です。

```bash
git push -u origin main
```

2回目以降は、ふつうにこれで送れます。

```bash
git push
```

## GitHub Pages で公開する

このテーマは Astro で動いているので、最終的には `dist/` を GitHub Pages に出します。

### 1. 先に本番ビルドを確認する

```bash
pnpm build
```

問題がなければ、ローカルで内容を確認します。

```bash
pnpm preview
```

### 2. GitHub Pages の設定を見る

GitHub のリポジトリ画面で次を開きます。

`Settings` -> `Pages`

### 3. 公開方法を選ぶ

おすすめは `GitHub Actions` です。

もし GitHub Actions で公開するなら:

1. `Settings` -> `Pages`
2. `Build and deployment` の `Source` を `GitHub Actions` にする
3. `main` に push したら自動で公開されるようにする

このリポジトリには今のところビルド用の CI はありますが、Pages へ送る専用 workflow はまだ別で必要です。

### 4. サイトの URL を合わせる

公開先が変わったら、次の 2 か所を確認します。

- [astro.config.ts](astro.config.ts)
- [src/config.ts](src/config.ts)

特に次が大事です。

- `themeConfig.site.website`
- `site`
- `base` を使う構成なら、その値

### 5. 反映される流れ

1. 記事や設定を編集する
2. `git add`
3. `git commit`
4. `git push`
5. GitHub Actions が動く
6. GitHub Pages に公開される

## 迷ったら

最初はこの順番でやれば大丈夫です。

1. `pnpm new "記事タイトル"` で記事を作る
2. 記事を書く
3. `pnpm dev` で確認する
4. `git status` で変更を見て、`git add` する
5. `git commit` して `git push` する
6. GitHub Pages の公開結果を確認する
