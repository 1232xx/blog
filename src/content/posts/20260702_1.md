# ITストラテジスト試験　半年合格ロードマップ

## ⚠️ 最初に重要な確認事項

**2026年度から試験制度が大きく変わります。**

- 試験方式：紙の試験 → **CBT方式**（パソコン受験）に移行
- 実施時期：従来の春期（4月）→ **前期（2026年11月頃）**に変更
- 科目名称：午前Ⅰ・午前Ⅱ → **科目A群**、午後Ⅰ・午後Ⅱ → **科目B群**
- 出題範囲・形式・試験時間自体は変更なし
- 申込みは科目A群・B群を同時予約、前期試験の申込みは1回のみ

つまり今から半年勉強する場合、**2026年11月頃の前期試験が本命**になります。最新情報はIPA公式サイトで随時確認してください。

---

## 結論：半年での合格は十分可能か

**可能です。ただし条件があります。**

- 合格率は例年14〜15%前後、必要勉強時間の目安は150〜200時間
- IT業界の実務経験者でも200時間程度かかるとされる難関試験
- 1日1時間ペース（週7時間）なら約半年〜7ヶ月で200時間に到達する計算
- 未経験者は経営戦略・IT戦略の知識がゼロからのスタートになるため、上記時間に加えてインプットの時間が追加で必要

**半年（約26週）で合格を狙うなら、平日1〜1.5時間＋休日3〜4時間（週10〜12時間、合計260〜310時間）を目安にすると安全圏です。**

## 試験の構成（変更後の名称も併記）

| 科目 | 旧名称 | 新名称（2026〜） | 内容 | 配点・形式 |
|---|---|---|---|---|
| 科目A-1 | 午前Ⅰ | 科目A群①相当 | 共通知識（応用情報レベル） | 多肢選択式・免除制度あり |
| 科目A-2 | 午前Ⅱ | 科目A群②相当 | ST専門知識 | 多肢選択式 |
| 科目B-1 | 午後Ⅰ | 科目B群①相当 | 事例解析・記述式 | 記述式 |
| 科目B-2 | 午後Ⅱ | 科目B群②相当 | 論文（経営戦略・IT戦略） | 論述式（2,000〜3,000字程度） |

**最大の壁は科目B-2（論文）です。** 多くの受験者がここで落ちます。早期から着手することが合格の鍵です。

## 全体ロードマップ（26週間・約半年）

```
Phase 1: 基礎固め（1〜6週）
        ↓
Phase 2: 専門知識・科目A対策（7〜12週）
        ↓
Phase 3: 科目B-1（記述式）対策（13〜18週）
        ↓
Phase 4: 科目B-2（論文）対策（19〜24週）
        ↓
Phase 5: 総仕上げ・模試（25〜26週）
```

---

### Phase 1：基礎固め（1〜6週目）

**目的**：経営戦略・IT戦略の全体像を掴む。ITストラテジストが何を問われる試験かを理解する。

| 週 | やること |
|---|---|
| 1週目 | 試験制度の理解（IPA公式サイト確認）、参考書1冊を通読 |
| 2〜3週目 | 経営戦略の基礎用語（SWOT分析・PPM・バランススコアカード等）を学習 |
| 4週目 | DX・IT戦略系の基礎知識（最近の出題トレンド）を学習 |
| 5〜6週目 | 科目A-2（専門知識）の過去問を1年分解いて現状の実力を把握 |

**使う教材**：ALL IN ONE パーフェクトマスター（後述）

---

### Phase 2：専門知識・科目A対策（7〜12週目）

**目的**：科目A-1・A-2を安定して合格点（60%）以上取れるようにする。

| 週 | やること |
|---|---|
| 7〜8週目 | 科目A-2の過去問演習を開始（181問の演習問題を回す） |
| 9〜10週目 | 間違えた問題のみ複数回復習。苦手分野を特定 |
| 11週目 | 科目A-1（応用情報レベル）の過去問を5年分、サラッと確認 |
| 12週目 | 科目A群の模擬試験を時間を計って実施 |

**ポイント**：実務経験があれば科目A-1は免除になる場合があるので、自分の免除資格を必ず確認してください。

---

### Phase 3：科目B-1（記述式）対策（13〜18週目）

**目的**：問題文を正確に読み取り、設問の意図に沿った解答を時間内に書く力をつける。

| 週 | やること |
|---|---|
| 13週目 | 解答テクニック（マーキング・下書きの仕方）を学ぶ |
| 14〜16週目 | 過去問演習を1日1問ペースで実施（18問分を目標） |
| 17週目 | 間違えた問題の解答プロセスを見直し、なぜそう書くべきだったかを言語化 |
| 18週目 | 時間を計って本番形式で演習（科目B-1は時間配分が命） |

**ポイント**：「問題文の中に答えがある」試験です。自分の意見ではなく、問題文に書かれた情報を根拠に解答する訓練が重要です。

---

### Phase 4：科目B-2（論文）対策（19〜24週目）★最重要

**目的**：経営戦略・IT戦略のテーマで2,000〜3,000字の論文を時間内に書き切る力をつける。

| 週 | やること |
|---|---|
| 19週目 | 論文の基本構成（起承転結・設問ア/イ/ウの型）を学ぶ |
| 20週目 | 自分の実務経験（または架空の事例）を2〜3パターン準備し「ネタ帳」を作る |
| 21〜22週目 | 過去問テーマで実際に論文を1本書いてみる（週2本ペース） |
| 23週目 | 書いた論文を自己採点・添削（可能なら通信講座等で他者添削を受ける） |
| 24週目 | 時間内（120分）で論文を書き切る練習を繰り返す |

**ポイント**：
- 多くの受験生が躓くのがこの論述対策で、早い段階からテーマについて考え、実際に書く練習を繰り返すことが合格への鍵
- 「事例を2つくらい用意して、様々な角度から論文に対応できるよう準備」という合格者の声が多数あり、**実務経験ネタを2〜3本仕込んでおき、どんな設問にも応用できる形にしておく**のが王道
- 週に2回ペースで論文構想〜執筆の演習を継続するのが効果的という合格者の声あり

---

### Phase 5：総仕上げ・模試（25〜26週目）

| 週 | やること |
|---|---|
| 25週目 | 科目A〜B全体を通した模擬試験を実施 |
| 26週目 | 苦手分野の最終確認、論文の型を最終チェック、試験当日のシミュレーション |

## 各フェーズで「これができてないとまずい」チェックリスト

### Phase 1終了時点（6週目）でできているべきこと
- [ ] SWOT分析・PPM・バランススコアカードなど主要な経営戦略フレームワークを人に説明できる
- [ ] 試験の科目構成（科目A群・B群）と配点を正確に把握している
- [ ] 科目A-2の過去問1年分を解いて、正答率が何%か把握している（50%以下でも今は問題なし、現状把握が目的）

**できていない場合**：Phase 2に進まず、もう1週間①の通読に充ててください。土台がないまま演習に入ると7〜12週目が崩れます。

### Phase 2終了時点（12週目）でできているべきこと
- [ ] 科目A-2の181問のうち、間違えた問題の正答率が80%以上になっている
- [ ] 科目A-1（免除でない場合）で安定して60%超えが取れている
- [ ] 模擬試験を時間内に解き切れている

**できていない場合**：13週目に進む前に1週間バッファを使って科目Aを固めてください。科目Aが不安定なまま科目Bに進むと、本番で科目Aから崩れて足切りになるリスクがあります。

### Phase 3終了時点（18週目）でできているべきこと
- [ ] 科目B-1の過去問18問を解き終えている
- [ ] 「問題文のどこを根拠にその解答を書いたか」を毎回説明できる
- [ ] 制限時間内（科目B-1は90分が目安）に最後まで解答欄を埋められる

**できていない場合**：これは特に注意です。19週目から論文対策に入る予定なので、ここで足踏みすると論文対策の時間が削られます。最低限「時間内に解き切る」ことだけは18週目で達成してください。

### Phase 4終了時点（24週目）でできているべきこと　★最重要
- [ ] 自分用の論文ネタが最低2パターン（できれば3パターン）完成している
- [ ] そのネタを使って実際に論文を5本以上書いた経験がある
- [ ] 120分で2,000〜3,000字を書き切れる（内容の完成度より「時間内に書き切る」が最優先）
- [ ] 第三者（添削サービス・知人・SNS等）に最低1回は論文を見てもらっている

**できていない場合**：ここが半年計画で唯一妥協できないポイントです。もし24週目時点でこれらが未達なら、25〜26週目は模試をやめて論文対策に全振りしてください。科目A・B-1がどれだけ仕上がっていても、論文がB+判定（不合格）だと総合不合格になります。

### Phase 5終了時点（試験前日）でできているべきこと
- [ ] 科目A・B-1・B-2すべてを通しで本番同様の時間配分で1回は実施している
- [ ] 論文のネタ帳を試験当日に持ち込める形（記憶 or メモ）で整理している
- [ ] CBT会場の場所・持ち物・受験時間をIPA公式サイトで最終確認している

## 各書籍のポイント（つまずきやすい点）

**①ALL IN ONE パーフェクトマスター**
最初は理解できなくて当然です。1周目は「こういう範囲があるんだ」と把握するだけで十分。完璧に覚えようとして1週目で止まる人が多いので注意。

**②用語集（電子書籍）**
通勤・寝る前などの隙間時間専用。机に向かって使う教材ではなく、スマホで1日5分だけ眺める使い方が正解。

**③重点対策（演習問題集）**
1回解いて終わりにしないこと。間違えた問題に印をつけて、その問題だけを3回転以上繰り返すのが正しい使い方。全部を毎回満遍なく解き直すのは非効率。

**④情報処理教科書**
科目B-1の章と科目B-2（論文）の章で使うタイミングが全く違う。13週目（B-1）と19週目以降（B-2）で別の章を使うことを意識する。

**⑤合格論文の書き方・事例集**
他人の論文をそのまま丸暗記しても意味がない。構成・展開の「型」を盗んで、自分の実務経験に当てはめて書き直す練習に使う。

## 半年で合格するための心構え

1. **論文対策（Phase 4）を先延ばしにしない**　最大の落とし穴です。20週目には最低1本書き始めてください。
2. **過去問は「解く」より「分析する」**　1問にかける時間を惜しまず、なぜその解答になるのかのロジックを理解することが重要です。
3. **科目Aで時間を使いすぎない**　実務経験があれば科目Aは比較的取りやすいので、早めに切り上げて科目Bに時間を割いてください。
4. **2026年度の試験制度変更を必ずIPA公式サイトで確認する**　CBT移行・日程変更の詳細は今後も更新される可能性があるため、申込み前に最新情報を確認してください。
